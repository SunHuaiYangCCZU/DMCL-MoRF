# DMCL-MoRF
A Contrastive Learning–Based Dual-Modal Fusion Framework of Protein Sequence and Structure for MoRF Prediction

Molecular Recognition Features (MoRFs) play a critical role in protein–protein interaction regulation, signal transduction, and various disease-related processes. Accurate identification of MoRF regions is essential for understanding protein functional mechanisms and facilitating drug discovery. Existing deep learning-based MoRF prediction methods often rely on single-source sequence information or lack effective modeling of three-dimensional structural features, limiting their ability to fully exploit complementary multi-source representations. To address these challenges, we propose a dual-modal fusion framework based on contrastive learning, termed DMCL-MoRF. The model integrates multi-source sequence representations generated by two pretrained protein language models, ProtT5 and ESM-2, and incorporates three-dimensional structural information predicted by AlphaFold2 through graph-based modeling, enabling collaborative learning of sequence and structural features. By introducing a cross-modal contrastive learning mechanism to align sequence and structure representations, and designing a Dual Convolution-Attention Fusion (DCAF) module to enhance dynamic integration of multi-source features, the model more effectively captures discriminative characteristics of MoRF regions. Experimental results demonstrate that DMCL-MoRF significantly outperforms existing state-of-the-art methods. On the benchmark dataset, DMCL-MoRF achieves an AUC of 0.9674 and an MCC of 0.7678, improving upon the best baseline by 3.89% and 8.07%, respectively, demonstrating its superior generalization ability and stability. Ablation studies further validate the effectiveness of the contrastive learning strategy and the proposed fusion mechanism. This work provides a unified modeling framework that integrates multi-source representations and structural information for MoRFs prediction.

# Description
The proposed DMCL-MoRF method is implemented in Python based on the PyTorch framework for predicting molecular recognition features (MoRFs) in proteins.  
DMCL-MoRF adopts a dual-branch network architecture and introduces a Bi-directional Local Cross Attention (BiLCrossAttention) module, which effectively fuses multi-source features and thereby significantly improves the accuracy of MoRF prediction.

# Datasets
DMCL-MoRF provides several pre-split datasets in FASTA format for model training and evaluation, including:
- `Train.fasta`:  
  The main training set of DMCL-MoRF, containing protein sequences used for model learning.
- `Training421.fasta`:  
  The classical TRAINING421 dataset, consisting of 421 protein sequences. It is one of the commonly used benchmark training sets in the MoRF prediction field.
- `Test1.fasta` and `Test2.fasta`:  
  Two newly constructed independent test sets, Test1 and Test2, used to systematically evaluate the generalization ability of DeepDPM under different data distributions. The MCC and AUC values reported in the paper are computed on these two datasets.
- `Test 419.fasta`:  
  The classical TEST419 dataset with 419 protein sequences, widely used for performance comparison in previous MoRF prediction methods. It serves as an important benchmark test set in this work.
- `Test 45.fasta` and `test49.fasta`:  
  Two additional independent test sets containing 45 and 49 protein sequences, respectively, which can be used to further verify the robustness of DMCL-MoRF on diverse independent data.
All of the above are protein-sequence-level datasets used for training and testing DMCL-MoRF on the MoRF prediction task.

# Feature
The Extract_features module contains three feature extraction scripts. By simply providing the path to the dataset, users can generate ESM-2, Prot-T5, and AlphaFold2 features.

# Model
The model-related code is stored in the `model` directory, which contains three scripts corresponding to the sequence branch, the structure branch, and the fusion module, respectively. The trained models are saved in the `save_model` directory.

# Test
The testing script is located in Test.py.
